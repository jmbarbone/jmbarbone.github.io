Dunning Kruger Effect

  r code   psychology datavis  Comments Word Count: 1655words Read Count: 8minutes

Note: I originally wrote this in February 2019.

Full code found here:

Introduction

In my Organization Psychology graduate class at West Chester University, one of our assigned readings (among others) for our week on emotions and moods was (Sheldon, Dunning, and Ames 2014).

The Dunning-Kruger effect is founding on the concept that an individual that lacks expertise will be more confident in their abilities than they really are, or overestimate their performance on a task. Yet experts may underestimate their own performance or abilities or be more accurate in their estimations. One thing we can derive from this is possibly that those with lower skill will overestimate their abilities while those more skilled will underestimate their abilities.

The following is in direct relation to an article by (Sheldon, Dunning, and Ames 2014). They report a significant relationship between an individual’s actual performance and the difference between their perceived ability and actual performance in three conditions (\(r_1 = -0.83\), \(p_1 < .001\); \(r_2 = -0.87\), \(p_2 < .001\); \(r_3 = -0.84\), \(p_3 < .001\)).

They also used these two graphs to representing their findings:

Figure 1. Overestimation of emotional intelligence (left panel) and performance on the Mayer-Salovey-Caruso Emotional Intelligence Test (MSCEIT; right panel) as a function of actual performance on the MSCEIT

We’ll go through and understand why this can be misleading and how to replicate the Dunning-Kruger effect with random data.

Set up

So let’s place with some random data and see what we get. First, let’s setup our RMD file and choose a specific randomization seed so we can come back to our results:

options(tidyverse.quiet = TRUE) ## silences warnings
library(tidyverse)
library(jordan) ## percentile_rank() | github.com/jmbarbone/jordan
theme_set(theme_minimal())

Random data

We’ll start by creating a data frame with two vectors of independent, random data. These will be our randomly assigned percentile ranks of actual and estimate’d performance.

To clarify, the calculation of percentile rank is as follows:

\[\text{PR}_i = \frac{c_\ell + 0.5 f_i}{N} \times 100%\]

Where \(c_\ell\) is the count of scores lower than the score of interest, \(f_i\) is the frequency of the score of interest, and \(N\) is the total number of scores. With this formula, our percentile ranks will always be 0 < \(PR_i\) < 100.

random_data <- tibble(actual = rnorm(1000),
                      estimate = rnorm(1000)) %>% 
  mutate_all(percentile_rank)

Already we’re seeing a trend very similar to that reported in the article. What we also notice is that there are bounds to the overestimation value as a factor of the individual’s actual performance. An individual that performs at the 99th percentile cannot overestimate their own performance (but can be accurate) - much like an individual in the lower percentiles would unlikely underestimate. These is additionally worsed by the use of a score derived in reference to others.

Adjusting random data

So now we’re going to take some data and use some rough estimates for means. We’ll use the results from the study of interest.

We’ll shape our normal distributions around the values found in the paper.

percentile_rank <- function(x) {
  floor(percent_rank(x) * 100)
}
adj_random <- tibble(actual = rnorm(1000, 50, sd = 20),
                     estimate = rnorm(1000, 75, sd = 15)) %>%
  mutate_all(percentile_rank)

More random data

So what if we repeated this several times but with smaller sample sets?

random_helper <- function(x) {
  set.seed(42 + x)
  tibble(actual = rnorm(60, 50, sd = 25),
         estimate = rnorm(60, 75, sd = 15)) %>%
    mutate_all(percentile_rank)
}

sev_random <- as.list(1:5) %>% 
  map(random_helper) %>% 
  bind_rows(.id = "id") %>% 
  mutate(id = as.numeric(id))

sev_bins <- sev_random %>% 
  group_by(id) %>% 
  mutate(difference = estimate - actual,
         bin = ntile(actual, 5) * 20 - 10) %>% 
  group_by(id, bin) %>% 
  summarise(n = n(),
            mean_est = mean(estimate),
            mean_diff = mean(difference))

ggplot(sev_bins, aes(x = bin, y = mean_est, col = factor(id))) +
  geom_point() +
  geom_line() +
  scale_color_discrete(name = "Randomization") +
  scale_y_continuous(limits = c(0, 99)) + 
  labs(x = "Actual",
       y = "Estimate")

So what if we actually run a correlation on these numbers? We’ll create a nested function and install the broom package to help tidy up our results.

library(broom)

run_correlations <- function(x, item_x, item_y) {
  corr_helper <- function(x, item_x, item_y) {
    formula <- str_c("~", item_x, "+", item_y, sep = " ")
    cor.test(eval(parse(text = formula)), data = x)
  }
  x %>% 
    nest(data = -id) %>% 
    mutate(corr = map(data, corr_helper, item_x, item_y),
           tidy = map(corr, tidy)) %>% 
    unnest(tidy) %>% 
    select_if(negate(is.list))
}

Interesting. When we calculated a correlation with the mean estimates we actually got a significant result from one of our random samples, id 3. Let’s pull that one out to look at it again.

ggplot(filter(sev_random, id == 3), aes(x = actual, y = estimate)) +
  geom_point(alpha = .2) +
  geom_point(data = filter(sev_bins, id == 3), aes(x = bin, y = mean_est)) +
  geom_line(data = filter(sev_bins, id == 3), aes(x = bin, y = mean_est))

So there you have it. A successful replication of this ‘effect’ with random data. For more information see articles by (Nuhfer et al. 2016).
